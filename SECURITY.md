# Security Policy

## Supported versions

Only the **latest release** receives security updates.
Older versions are not patched — please upgrade to the latest release.

| Version | Supported |
|---|---|
| Latest | ✅ |
| Older | ❌ |

## Reporting a vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

Public disclosure before a fix is available puts all users at risk.

### How to report

Send an email to: **security@sportpulse.app**
*(TODO: replace with your actual security contact address)*

Please include:
1. **Description** — clear explanation of the vulnerability
2. **Steps to reproduce** — minimal reproduction steps or proof-of-concept
3. **Potential impact** — what an attacker could achieve
4. **Affected versions** — which versions are affected (if known)

### What to expect

- **Acknowledgment**: within **48 hours**
- **Status update**: within **7 days** (confirmed, investigating, or declined)
- **Fix timeline**: depends on severity — critical issues are prioritised

We follow responsible disclosure: once a fix is released, we'll credit the reporter (unless they prefer to remain anonymous) in the GitHub release notes.
